# /compact 引き継ぎプロンプト生成

このスキルは「いまの会話を /compact した直後の自分（=文脈ゼロの他人）が、迷わず作業を再開できる」プロンプトを 1 本生成する。**実装作業はしない。生成のみ。**

## 出力ゴール

最終的にユーザーに渡すのは、以下のテンプレートに沿った **1 つのコードブロック**（コピペで `/compact <ここ>` に貼れる形）。各セクションは「具体・固有名詞・パス・コマンド」を必ず残し、抽象語（「適宜」「など」「いい感じに」）を禁止する。

````
# Context Handoff (compact)

## 1. ゴール / Why
<このセッションが達成しようとしている最終ゴールを 1-3 行。なぜやっているか（背景・依頼者・締切）も含める>

## 2. 現在地 / Status
- フェーズ: <調査 / 計画 / 実装 / レビュー / デバッグ のどれか>
- 直近の判断: <最後に下した方針判断と理由>
- 直近の手詰まり / 仮説: <あれば>

## 3. これまでにやったこと / Done
- <時系列または論点別に箇条書き。「何を読んだ・何を実行した・何が分かった」を事実ベースで>
- <重要な発見・観測した挙動・エラーメッセージ原文も残す>

## 4. 残タスク / Next Actions
- [ ] <次の一手。具体的な動詞 + 対象ファイル/コマンドで書く>
- [ ] <...>
（優先順に並べる。依存関係があれば明記）

## 5. 環境 / Anchors
- repo: <絶対パス>
- branch: <現在ブランチ> (base: <main など>)
- 触ったファイル: 
  - `path/to/file.ts` — <どう触ったか>
- 関連コマンド / 再現手順:
  ```
  <そのまま叩けるコマンド>
  ```
- 外部リソース: <PR URL / Notion / Slack スレッド / ログクエリ など>

## 6. 制約 / 決め事
- <ユーザーから受けた指示・禁止事項・スタイル>
- <技術的制約・既存の設計判断>

## 7. 未解決の論点 / Open Questions
- <意思決定待ち。ユーザー確認が必要なものはここに集約>

## 8. 再開時の最初の一手
<compact 後の自分が読み終わって最初に実行すべき具体アクションを 1 行>
````

## 生成手順

1. **会話履歴を自分で走査する**（このスキル本体に渡されている会話文脈）。直近の TodoWrite / Plan / Edit / Bash 実行・ユーザー指示・advisor 助言を拾う。
2. **不足情報は最小限のツールで補完**:
   - `git status -s` / `git branch --show-current` / `git log --oneline -5` で環境セクションを正確に
   - 必要なら触ったファイルの Read（全文ではなく該当箇所のみ）
   - 余計なファイル探索はしない。**新規調査は禁止**（あくまで引き継ぎプロンプト作成）
3. **テンプレートを埋める**。書けない欄は「（情報なし）」と明記して残す（沈黙させない）。
4. **品質チェック**:
   - 抽象語禁止リスト: 「適宜」「いい感じに」「など」「適切に」「必要に応じて」→ あれば具体化
   - Next Actions が「動詞 + 対象」になっているか
   - パス・ブランチ・コマンドが省略されていないか
   - 8 セクション全部埋まっているか
5. **ユーザーに返す**: コードブロック 1 本のみ。前置きは「以下を `/compact` に渡してください」程度の 1 行に留める。

## 振る舞いの原則

- **要約ではなく引き継ぎ**: 何が起きたかの説明ではなく、「次の自分が動けるための作業仕様書」を書く
- **冗長より具体**: 文章を短くするより、ファイルパス・コマンド・エラー原文を残す方を優先
- **未来形より命令形**: 「〜する予定」ではなく「<file> の L42 を X に変更する」
- **memory には書かない**: これはセッション継続のためのプロンプトであり永続記憶ではない
- **ユーザーが「これも入れて」と追加情報を渡したら、該当セクションに追記して再出力**
