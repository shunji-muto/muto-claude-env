# PRマージとブランチ削除

現在チェックアウト中のブランチをベースブランチへマージし、マージ後に不要となるブランチを削除します。

## 実行手順

### 1. 現在ブランチとベースブランチの確認

```bash
# リポジトリルートへ移動
cd "$(git rev-parse --show-toplevel)"

# 現在ブランチを取得
git rev-parse --abbrev-ref HEAD

# ベースブランチを解決（origin/HEAD から自動検出、デフォルトは main）
git fetch origin --prune
```

**チェック:**
- 現在ブランチがベースブランチ（`main` または `master`）の場合は処理を中止
- ベースブランチは `origin/HEAD` から自動解決、見つからなければ `main` を使用

### 2. リモート同期とpush確認

```bash
# ブランチがリモートに存在しない場合はpush
git push -u origin <現在のブランチ名>
```

### 3. PR検出または作成

```bash
# 既存PRを検出
gh pr view --json number -q .number

# PRが存在しない場合は作成
gh pr create \
  --title "<ブランチ名またはカスタムタイトル>" \
  --body "Auto-created PR for <ブランチ名> -> <ベースブランチ>" \
  --base <ベースブランチ> \
  --head <現在のブランチ名>
```

### 4. PRマージ実行

```bash
# PRをスカッシュマージ（ブランチ削除オプション付き）
gh pr merge <PR番号> \
  --squash \
  --delete-branch
```

> 🚨 **重要: マージ方式は必ず `--squash` を使用すること。**
> このリポジトリはスカッシュマージのみ許可されています。
> `--merge`（通常マージ）や `--rebase`（リベースマージ）は使用禁止です。

**自動マージの使用:**
```bash
# ステータスチェック完了後に自動マージ（スカッシュのみ）
gh pr merge <PR番号> --squash --delete-branch --auto
```

### 5. ブランチ削除

```bash
# ベースブランチへチェックアウト
git checkout <ベースブランチ>

# ローカルブランチを削除
git branch -D <マージ済みブランチ名>

# リモートの参照を整理
git remote prune origin
```

### 6. 結果出力

```bash
# PR情報を表示
gh pr view <PR番号> --json url,state,mergedAt,mergeCommit
```

**出力内容:**
- PR URL
- PR状態（MERGED）
- マージ時刻
- マージコミットのSHA

## エラーハンドリング

### ステータスチェック未完了

CIやテストが完了していない場合：
- `--auto` オプションを設定して自動マージを有効化
- ステータスチェック完了後に再実行

### マージコンフリクト

コンフリクトが発生した場合：
1. ベースブランチを現在のブランチにマージ
2. コンフリクトを解決
3. `git push` で更新
4. 再度マージを実行

## 前提条件

- リポジトリがGitHub上の `origin` に設定されていること
- `gh` CLIがインストール・認証済みであること
- 現在のブランチがベースブランチでないこと
- マージ権限があること

## 注意事項

- **マージ方式は必ず `--squash`（スカッシュマージ）を使用すること。このリポジトリはスカッシュマージのみ許可されており、`--merge` や `--rebase` は使用禁止**
- `--delete-branch` によりリモート/ローカルのブランチが削除される
- マージ後は自動的にベースブランチへチェックアウトされる
- 削除されたブランチは復元できないため、慎重に実行
