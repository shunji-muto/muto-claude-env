## 使用方法

### 1. UIの改善提案

既存のUIに対して、関連するTipsを引用しながら改善点を提案します。

**手順:**
1. 対象のUI/画面を確認（コード読み込み or スクリーンショット確認）
2. 問題点を特定
3. 関連カテゴリのTipsファイルを参照
4. 具体的な改善案を提示

**出力フォーマット:**
```markdown
## 現状の問題点

### 問題1: [問題の要約]
- **状況**: [具体的な状況]
- **関連Tips**: #XX [Tips名]（カテゴリ: XX）
- **引用**: > [Tipsの内容を引用]
- **改善案**: [具体的な改善提案]
```

### 2. 新規UI設計のアドバイス

新しいUIを設計する際に、参考になるTipsをカテゴリ別に提示します。

**手順:**
1. 要件を確認
2. 関連カテゴリを特定
3. 該当カテゴリのTipsファイルから推奨項目を抽出
4. 設計の指針を提示

### 3. デザインレビュー

UIのデザインレビュー時に、Tipsに基づいた客観的なフィードバックを提供します。

## 使用例

### 例1: フォーム画面の改善

```
User: このログインフォームを改善したい

Claude: フォームを確認します...

## 現状の問題点

### 問題1: エラーメッセージが分かりにくい
- **状況**: 「入力エラー」とだけ表示される
- **関連Tips**: #55 エラー表示は建設的にする（05-feedback-navigation）
- **引用**: > エラー発生時にはわかりやすいメッセージによって
  ユーザーの理解と行動を手助けする。何が起きたのか的確に知らせ、
  そこからどうすればよいのかを示す。
- **改善案**: 「メールアドレスの形式が正しくありません。
  例: example@email.com」のように具体的に示す
```

```
## 推奨UX心理学コンセプト

### UX心理学（09-ux-psychology）
- #2 アンカー効果: 最も高いプランを最初に表示して
  他のプランをお得に感じさせる
- #8 おとり効果: 中間プランをおとりにして
  上位プランの価値を際立たせる
- #9 デフォルト効果: おすすめプランをデフォルトで
  選択状態にする

- #1 シンプルにする: プラン比較を3つ程度に絞る

```

## 注意事項

1. **Tipsは絶対ではない**: 文脈によっては適用しない方が良い場合もある
2. **トレードオフを考慮**: 複数のTipsが矛盾する場合は優先度を判断
3. **ユーザーテストを重視**: Tipsに従っても実際の効果は検証が必要
4. **原典を参照**: より詳しい説明はカテゴリ別ファイルを参照
